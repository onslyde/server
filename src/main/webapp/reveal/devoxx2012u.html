Web Sockets

Every HTTP request sent from the browser includes headers, whether you want them or not.

Uncompressed request and response headers can vary in size from 200 bytes to over 2K.

WebSockets give you minimal overhead with full duplex communication through a single socket.


supports_websocket:function () {
    try {
        return 'WebSocket' in window && window['WebSocket'] !== null;
    } catch (e) {
        return false;
    }
}

Web Storage

Today, you have two well-supported mechanisms for storing data on the client: cookies and Web Storage.

Cookies are much different than Web Storage and implicitly send data back to the server upon each request through HTTP headers.

Cookies only give you 4kb of max storage per cookie

Geolocation

(not HTML5)

The Geolocation API provides scripted access to geographical location information associated with the hosting device.

  • Geofencing
  • Geocoding
  • General tracking

The API itself is device agnostic; it doesn’t care how the browser determines location.

  • Wifi
  • GPS
  • (User Entered) Zip Code

//geolocation cannot be accessed with dot notation in iOS5... will prevent page caching

    } catch (e) {
        return false;

Web Workers

The Web Workers interface spawns real OS-level threads, allowing for data to be passed back and forth between any given threads (or worker)

You cannot access components unsafe to threads or the DOM, and you have to pass specific data in and out of a thread through serialized objects.

Client vs. Server UI

Between 2008-2009 HTML generation on the back-end broke down

Progressive enhancement with AJAX

This forced developers to create better APIs

...and more efficient ways of delivering data via JSON and XML

Client side generation (PROs):

  • Reduces server load
  • JSON uses less bandwidth than presentation ready HTML
  • CDN Bandwidth (jQuery, BackBone, etc...)
  • Offline capability \o/

Client side generation (CONs):

  • Can reduce performance of device/browser
  • Parsing JSON and generating HTML uses more memory
  • Hardware access === battery drain

The most notable client side con is security

  • Web Storage is widely supported but not secure

Don't end up with a huge mess!

  • 2012-2013 is all about hybrid
    • Microsoft, Blackberry, PhoneGap, Appcelerator...
  • Dealing with separate code bases takes special care
  • JavaScript MV* frameworks help with this

Front-End Performance

  • Developers inherently care about performance
  • Product, Sales, and Marketing folks don’t care as much
  • Performance impacts the bottom line of the business
  • When speed drops off, the performance of the business drops off
